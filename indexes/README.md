# Indexes

Indexes route questions to the right project files.

- `task-router.md`: first file to read for a user request.
- `rules-map.md`: high-level subsystem overview.
- `subsystem-index.md`: summary of rule file groups.
- `term-glossary.md`: shared terminology and aliases.
- `page-reference-index.md`: source page reference map.
- `manifest.yaml`: structured file metadata. `rules` entries point to committed draft summaries, `mapped_rules` entries are non-authoritative source-location targets, and `metadata.status_legend` defines the allowed coverage/status labels.
