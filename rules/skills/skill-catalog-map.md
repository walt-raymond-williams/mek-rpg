# Skill Catalog Map

## Purpose

Use this routing aid to locate the right skill source area, record skill metadata correctly, and choose the appropriate rule summary for action resolution. It does not reproduce the master skill list or individual skill descriptions.

## When to Use

- A character sheet needs exact skill name, subskill, specialty, linked attributes, target number, complexity, or tier notation.
- A player asks which skill fits a proposed action or background role.
- A build uses Skill Fields, mandatory language/perception purchases, vehicle skills, technical skills, or social/investigation skills.
- A GM needs a source page pointer for exact skill descriptions and modifiers.

## Do Not Use For

- Copying the master skill list, linked-attribute table, TN/complexity table, or full skill descriptions.
- Replacing `rules/core/skill-checks.md` for live action resolution.
- Replacing personal combat, vehicle, equipment, medical, or campaign summaries when a specialized procedure exists.
- Deriving Classic BattleTech unit procedures or tactical modifiers inside MEK-RPG.

## Basic Procedure

1. Identify the proposed action or build need in plain language.
2. Check whether a specialized subsystem summary owns the procedure first: personal combat, equipment, vehicles, campaign consequences, or tactical handoff.
3. If a character skill is needed, inspect the skill catalog source pages for exact skill name, subskill, linked attributes, TN, complexity, tier status, and any source guidance.
4. Record subskills after a slash and specialties in parentheses. Do not treat a specialty as a separate skill.
5. For tiered skills, record the correct tier metadata for the character's current level.
6. For action resolution, return to `rules/core/skill-checks.md` and `rules/core/basic-action-resolution.md` after choosing the skill.
7. If the action enters full tactical BattleTech, use `gm/switch-to-classic-battletech.md` instead of forcing the skill catalog to adjudicate unit combat.

## Practical GM Guidance

- Let the fictional approach choose the skill. The same obstacle can route to Security Systems, Technician, Computers, Negotiation, Acting, Forgery, brute force, or another approach.
- Use Career, Interest, and Science only when a more specific skill does not cover the task.
- Vehicle and weapon skills often require subskills. Record the operating platform, not just the root skill.
- Language usually records fluency rather than demanding rolls. Check trait interactions when reading or writing matters.
- Training matters for advancement, especially Advanced skills. Use `rules/character-creation/xp-advancement.md` for downtime improvement.

## Common Edge Cases

- A character can be trained in one subskill and untrained in another under the same root skill.
- A specialty gives a narrow focus and may create a penalty outside that focus; it is not a broad permission slip.
- Tiered skills can change linked attributes and complexity when the active level crosses the tier threshold.
- Tactical sourcebooks may use modifiers differently from A Time of War action checks; vehicle summaries note the inversion where relevant.
- Constant Perception or Investigation rolling can flatten play. Ask for rolls when failure matters or when the character's approach would reveal something specific.
