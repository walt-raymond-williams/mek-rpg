# Action Checks

## Purpose

Action Checks are the basic procedure for resolving character actions that can fail and where the result matters to the scene. They establish whether the GM should call for an Attribute Check, a Skill Check, or an opposed procedure.

## When to Use

- A character attempts something uncertain under pressure, danger, limited time, or meaningful consequences.
- The scene needs a roll to determine success, failure, or quality of outcome.
- The GM needs to choose between raw capability, trained skill, or direct opposition.

## Do Not Use For

- Mundane actions, ordinary conversation, or outcomes where failure would not change anything important.
- Full tactical BattleTech combat where facing, heat, armor locations, weapon ranges, or unit turns matter.
- Character creation, advancement, equipment purchase, injury recovery, or other subsystem procedures that have their own summaries.

## Basic Procedure

1. State the character's intent and what changes on success or failure.
2. Decide whether a trained Skill applies. If so, use `rules/core/skill-checks.md`.
3. If no trained Skill applies, or the character is attempting a Skill untrained, use `rules/core/attribute-checks.md`.
4. If another character is actively contesting the outcome, use `rules/core/opposed-actions.md`.
5. Resolve the roll with `rules/core/basic-action-resolution.md`, including modifiers and margin.

## Practical GM Guidance

Call for one roll only when the table needs uncertainty. Before the roll, tell the player the action being tested, the relevant basis for the check, and the immediate stakes. If a failed roll would only stall the scene, convert failure into a cost, delay, complication, partial information, or increased risk.

## Common Edge Cases

- If a character has no relevant Skill, do not improvise a Skill bonus; route the action through the appropriate Attribute Check.
- If a character's trained Skill only covers part of a broader task, use the Skill for that part and call for later checks only when new uncertainty appears.
- If two characters both roll against each other, success alone is not enough; compare margins through the opposed-action procedure.
- If a player wants to spend Edge, read `rules/core/edge.md` before finalizing the roll.
