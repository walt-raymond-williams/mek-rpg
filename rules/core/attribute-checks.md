# Attribute Checks

## Purpose

Attribute Checks resolve actions using raw character capability when no trained Skill applies, or when a character attempts a Skill without training. They use one or two Attributes selected from the situation or from the untrained Skill's linked Attributes.

## When to Use

- The action directly tests physical, mental, social, or reflexive capability rather than learned technique.
- No listed trained Skill covers the action.
- The character lacks the relevant Skill, including level 0, but the GM allows an untrained attempt.
- An untrained Skill identifies linked Attributes that can stand in for the missing training.

## Do Not Use For

- Actions covered by a trained Skill the character possesses.
- Opposed contests without reading the opposed-action procedure.
- Full combat attacks, injury effects, recovery, equipment procedures, or tactical BattleTech play.

## Basic Procedure

1. Decide whether the check uses one Attribute or two Attributes.
2. For a normal Attribute Check, choose the Attribute or Attributes that best represent the action.
3. For an untrained Skill attempt, use that Skill's linked Attribute or linked Attributes.
4. Use the fixed Attribute Check target number for a single-Attribute or double-Attribute check; see `rules/core/basic-action-resolution.md` for the target-number table reference.
5. Roll 2D6, add the relevant Attribute score or combined Attribute scores, then add any situational modifiers the GM applies.
6. Compare the final result to the target number and interpret margin through `rules/core/basic-action-resolution.md`.

## Practical GM Guidance

Use Attribute Checks when the fiction is about raw capacity: holding weight, reacting quickly, remembering under pressure, forcing composure, or trying an untrained task. Be explicit about why the check is one Attribute or two; this keeps untrained attempts consistent and prevents them from outperforming trained characters too easily.

## Common Edge Cases

- Untrained Skill use follows the Attribute Check target numbers, not the trained Skill's lower target number.
- Attribute Checks do not use linked-Attribute modifiers in the same way trained Skills do; they add the applicable Attribute score or scores directly.
- When a Skill has two linked Attributes and the character is untrained, treat it as a double-Attribute check unless the GM has a specific reason to narrow the task.
- If the source table value or linked Attribute detail is needed at the table, inspect the cited source pages rather than reconstructing the table from memory.
