# Task Checks
## Purpose
What this rule is for.
## When to Use
Use this rule when a character attempts an uncertain action against difficulty, circumstances, or time pressure.
## Do Not Use For
Cases where another subsystem should be used.
## Basic Procedure
1. Define the action and consequence.
2. Select the relevant ability, skill, or task basis from the verified summary.
3. Apply modifiers, roll, and interpret success or failure.
## Practical GM Guidance
Use one clear roll for one clear obstacle. Tell the player what success changes.
## Common Edge Cases
- Edge case: Another character actively resists, so use opposed checks.
- Edge case: Tactical BattleMech action may require Classic BattleTech instead.
