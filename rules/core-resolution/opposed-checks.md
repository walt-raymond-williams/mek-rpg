# Opposed Checks
## Purpose
What this rule is for.
## When to Use
Use this rule when two characters or forces directly contest an outcome.
## Do Not Use For
Cases where another subsystem should be used.
## Basic Procedure
1. Identify both sides and what each side wants.
2. Determine the relevant checks for each side.
3. Compare results and apply the winner's outcome and any margin effects.
## Practical GM Guidance
Make the stakes symmetrical when possible: what happens if either side wins should be clear.
## Common Edge Cases
- Edge case: One side is passive, so a normal task check may be enough.
- Edge case: Group contests may need a simplified lead actor or assistance procedure.
