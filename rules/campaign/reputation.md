# Reputation
## Purpose
What this rule is for.
## When to Use
Use this rule when public standing, faction opinion, or past actions affect opportunities.
## Do Not Use For
Cases where another subsystem should be used.
## Basic Procedure
1. Identify the relevant audience or faction.
2. Apply verified reputation effects if available.
3. Update access, prices, trust, opposition, or mission offers.
## Practical GM Guidance
Make reputation specific by faction and recent events.
## Common Edge Cases
- Edge case: Secret actions may not affect public reputation yet.
- Edge case: Conflicting faction reputations can both be true.
