# Contracts
## Purpose
What this rule is for.
## When to Use
Use this rule when missions, employment, payment, salvage, or obligations matter.
## Do Not Use For
Cases where another subsystem should be used.
## Basic Procedure
1. Define employer, objective, payment, risks, and constraints.
2. Apply verified contract or negotiation rules if available.
3. Track completion, breach, salvage, and reputation effects.
## Practical GM Guidance
Contracts should create choices: money, loyalty, risk, salvage, reputation, or time.
## Common Edge Cases
- Edge case: MekHQ may track force contracts and logistics.
- Edge case: Hidden employer motives can change aftermath.
