# Action And Movement

## Purpose

Action and movement rules define what a character can do during the Action Phase of a personal-combat turn and how far that character can move.

## When to Use

- A personal-combat turn reaches a character's place in initiative order.
- A character wants to move, attack, aim, recover, operate equipment, speak, take cover, or use a skill while under fire.
- Terrain, encumbrance, movement mode, or previous movement affects an attack.
- The GM needs to know whether a declared action is incidental, simple, complex, or movement.

## Do Not Use For

- Noncombat scenes where five-second turns are not useful.
- Full tactical BattleTech movement, facing, heat, unit initiative, or armor handling.
- Exact movement rates, terrain costs, encumbrance thresholds, or action catalog lookup from memory; use the cited source pages when precision matters.

## Basic Procedure

1. On the character's initiative, ask what the character is doing and how the character is moving.
2. Classify non-movement actions as incidental, simple, or complex.
3. Classify the movement mode and count it against the character's action budget for the turn.
4. Allow incidental actions within the source limit, unless the character is stunned, unconscious, immobilized, or otherwise prevented from acting.
5. Allow either the permitted number of simple actions or one complex action, following the source action-complexity limits.
6. Require one movement declaration each turn for any mobile conscious character. If none is declared, treat the character as stationary.
7. Calculate movement from the character's movement rates, selected movement mode, terrain, maneuvers, and encumbrance.
8. Resolve attacks, skill checks, and other action checks immediately unless the action is extended.
9. Track the movement mode because it can modify attacks made by or against the character.

## Practical GM Guidance

Keep the Action Phase crisp: intent first, movement second, roll only when failure matters. A player should know whether they can both move and shoot, sprint and do nothing else, or spend the turn on a complex task.

When exact movement matters, use the source movement tables. When it does not, convert the movement rules into clear choices: reach cover, stay exposed and shoot, sprint to the objective, crawl quietly, climb, swim, or recover.

## Common Edge Cases

- A character may use only one movement type per turn.
- Running uses more action attention than walking; sprinting consumes the turn and creates fatigue.
- Climbing and swimming become harder if the character lacks the relevant skill.
- Crawling and going prone can make a ranged target harder to hit but constrain movement.
- Encumbrance can increase movement costs and add fatigue under the source conditions.
- Until a character acts in the current turn, use the character's previous movement rate for movement-related attack modifiers unless the GM has a reason to reduce it.
- Squad and team initiative require unit cohesion; separated members may need to act as a separate unit.
