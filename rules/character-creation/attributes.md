# Attributes

## Purpose

Use this summary to assign, interpret, and later improve a character's core attributes during character creation and advancement.

## When to Use

- A player is assigning attributes during Life Module cleanup or points-only creation.
- A character must meet an attribute prerequisite for a module, skill field, trait, or later advancement.
- The GM needs to understand how attributes interact with linked skills and derived sheet values.
- A character wants to improve an attribute after play begins.

## Do Not Use For

- Resolving raw attribute checks in play. Use `rules/core/attribute-checks.md`.
- Reproducing the XP cost table, age table, phenotype limits, or sample-character math. Use the cited private source pages.
- Trait-specific attribute maximum changes without checking the exact trait source.
- Tactical pilot or unit ratings. Use vehicle and tactical handoff summaries when source-reviewed.

## Basic Procedure

1. Record the eight attributes on the character sheet: Strength, Body, Reflexes, Dexterity, Intelligence, Willpower, Charisma, and Edge.
2. During Life Module creation, add all module-assigned and flexible XP to the appropriate attributes, then determine final scores after all allocation and cleanup are complete.
3. During points-only creation, buy attribute scores directly from the XP pool while checking minimums, maximums, phenotype limits, and prerequisites.
4. Treat the normal minimum attribute value as 1 unless the character's affiliation, phenotype, module, or desired field requires otherwise.
5. Check whether traits such as phenotype or exceptional attribute alter maximums before finalizing the sheet.
6. Record linked-attribute modifiers and derived values only after final attribute scores are known.
7. For post-creation advancement, spend XP and justify practice, training, or campaign events that plausibly support the increase.

## Practical GM Guidance

- Attributes define broad capability; skills usually decide trained action. Do not overuse raw attribute checks when a skill summary applies.
- Check prerequisites late but before play starts. A Life Module character may temporarily fall short during the build as long as the finished sheet is legal.
- Character concept should lead attribute emphasis. A scout, MechWarrior, diplomat, medic, and technician should not all optimize the same way.
- If the player has leftover XP that does not reach the next attribute score, record it on the sheet as progress toward future advancement.
- Edge is both an attribute and a table-facing resource. Use `rules/core/edge.md` for spending and recovery guidance.

## Common Edge Cases

- Age can affect attributes after character creation. Use `rules/character-creation/xp-advancement.md` and the cited advancement pages.
- The Edge attribute follows normal purchase logic for improvement, but in-play Edge use and recovery are separate procedures.
- Phenotype and exceptional-attribute effects can change caps. Do not infer maximums from ordinary humans when the character is Clan-bred, augmented, or otherwise unusual.
- A module or field prerequisite can force a minimum attribute above the player's preferred concept. Resolve this before play rather than during the first relevant roll.
- Attribute improvements after play should not be instant bookkeeping if the campaign fiction offers no time or reason for improvement.
