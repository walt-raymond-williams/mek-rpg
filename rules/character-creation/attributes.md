# Attributes
## Purpose
What this rule is for.
## When to Use
Use this rule when assigning, interpreting, or advancing character attributes.
## Do Not Use For
Cases where another subsystem should be used.
## Basic Procedure
1. Identify the attribute relevant to the action or build step.
2. Apply the verified creation or advancement procedure.
3. Note how the attribute interacts with skills and derived values.
## Practical GM Guidance
Use attributes to describe broad capability, not every small moment of play.
## Common Edge Cases
- Edge case: A skill may be more important than a raw attribute.
- Edge case: Derived values need source review.
