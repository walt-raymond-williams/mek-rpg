# Traits
## Purpose
What this rule is for.
## When to Use
Use this rule when a character has advantages, disadvantages, resources, or special qualities.
## Do Not Use For
Cases where another subsystem should be used.
## Basic Procedure
1. Identify the trait and its verified effect.
2. Apply it when the trigger condition is met.
3. Record any campaign-facing consequence or limitation.
## Practical GM Guidance
Use traits to create recurring choices and complications, not only static bonuses.
## Common Edge Cases
- Edge case: Trait text may require exact source review.
- Edge case: Multiple traits may interact in a way that needs adjudication.
