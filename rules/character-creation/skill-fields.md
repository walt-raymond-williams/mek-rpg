# Skill Fields

## Purpose

Use this summary to understand and route Skill Fields during character creation. A Skill Field is a bundled training package that supports a recognizable background, profession, military role, technical specialty, or Clan role.

## When to Use

- A Life Module or points-only character selects a field such as military training, professional education, technical training, piloting, intelligence, policing, or Clan warrior training.
- A sheet review needs to confirm field prerequisites, included skills, and role coherence.
- A player wants a character whose skills match a profession without choosing every skill from scratch.
- The GM needs to find the private source pages for the exact field entries.

## Do Not Use For

- Copying the master field lists or exact field contents into committed files.
- Ignoring prerequisites for attributes, traits, affiliation, prior fields, rank, or phenotype.
- Treating a field name as permission to use a skill the character did not actually receive or buy.
- Resolving a trained action in play. Use `rules/core/skill-checks.md` after the correct skill is identified.

## Basic Procedure

1. Confirm the character creation method. Life Modules may grant or require fields; points-only characters may also buy fields within the source limits.
2. Identify the field category and concept: civilian, intelligence/police, military, or Clan military.
3. Check prerequisites before finalizing the sheet. Prerequisites can include attributes, traits, prior fields, affiliation, rank, phenotype, or disallowed traits.
4. Record the exact field name and source page reference on the working sheet or build notes.
5. For every skill received or bought through the field, record the exact skill name, required subskill, level or XP, linked attributes, TN/complexity, and any specialty only if approved.
6. If using points-only field purchases, buy all field skills together according to the source procedure and apply any field rebate or limit from the source pages.
7. After all fields and module choices are selected, verify that the final character still satisfies mandatory language, perception, attribute, trait, and field requirements.
8. Convert the field into background: institution, unit, apprenticeship, academy, guild, service branch, caste training, or professional network.

## Practical GM Guidance

- Use Skill Fields to answer "what was this character trained to do?" before looking at individual skill entries.
- The field name should create campaign hooks. A police tactical officer, DropShip pilot, HPG technician, Clan Elemental, or MechWarrior field implies contacts, duties, enemies, restrictions, and expected missions.
- Do not let field bundles replace sheet review. The sheet still needs exact subskills and active levels.
- For quick NPCs, fields can guide role competence, but exact skill entries remain a source lookup if the NPC will roll or enter combat.
- If a desired concept barely misses a prerequisite, decide whether the campaign supports a GM exception before play begins.

## Common Edge Cases

- Points-only characters have limits on field selection and must still purchase field skills under the source procedure.
- Some field prerequisites depend on earlier field choices. Do not treat all fields as independent shopping options.
- Clan fields can depend on affiliation and phenotype. Check `rules/character-creation/traits.md` and the trait catalog map when a trueborn or Clan-trained character is involved.
- A field can include skills with mandatory subskills. Record the subskill as part of the skill entry.
- Field skill lists are dense catalog material; use the private source pages for exact contents rather than copying lists into campaign notes.
