# Purchasing Character Elements

## Purpose

Use this summary to route the final spending and cleanup phase of character creation: determining active levels, meeting minimums, resolving opposed traits, optimizing leftover XP, optionally buying additional XP through negative traits, and using the points-only method.

## When to Use

- A Life Module character has accumulated XP in attributes, traits, and skills and needs final active values.
- A points-only character is buying attributes, traits, skills, and fields directly.
- A player wants to use negative traits to gain additional creation XP.
- A sheet has leftover XP, partial traits, partial skills, or conflicting/opposed traits.
- A GM needs to check whether a finished build is legal enough for play.

## Do Not Use For

- Reproducing the XP cost table, opposed-trait table, worked examples, field lists, trait catalog, or skill catalog.
- Awarding or spending XP after play begins. Use `rules/character-creation/xp-advancement.md` and `rules/campaign/advancement.md`.
- Treating negative traits as pure accounting. They must fit the character and campaign.
- Overriding phenotype, attribute maximum, trait maximum, field prerequisite, or training restrictions without a GM ruling.

## Basic Procedure

1. Gather all XP assigned by modules, fields, direct purchases, and remaining pool.
2. Determine active levels only from fully purchased thresholds. Store incomplete XP toward future advancement without treating the next value as active.
3. Check character minimums. Attributes cannot remain below the minimum; skills below the lowest active level drop or must be reconciled under the source procedure; trait minimums and maximums are trait-specific.
4. Check character maximums for attributes, traits, and skills. Recover or reassign unsupported excess XP only through the source cleanup process.
5. Resolve opposed traits before optional optimization. When traits or character features oppose one another, use the source procedure and record the remaining active trait or cancellation.
6. Decide whether to optimize leftover points. Optimization is a creation-phase cleanup option that shaves unusable excess XP from categories and returns it to the pool under the source limits.
7. If the table allows optional negative-trait XP, add or deepen negative traits only after optimization, only to fully attained negative levels, and only within the source cap based on the starting XP allotment.
8. For points-only creation, buy attributes, traits, fields, skills, mandatory language/perception entries, specialties, and leftover XP directly while checking all prerequisites and limits.
9. Record unresolved build questions on the sheet instead of silently assuming an answer.

## Practical GM Guidance

- Run cleanup slowly and visibly. Most build mistakes come from active values being treated as higher than the XP actually supports.
- Negative traits should produce future scenes, restrictions, or complications. If no one can describe how the drawback matters, reject or revise it.
- Points-only creation is efficient for NPCs and custom concepts, but it is easier to miss background logic. Add a short biography or hook pass after the math.
- Do not use optimization during play as a general refund mechanic. It belongs to character creation unless the GM makes a campaign-specific exception.
- Keep exact arithmetic near the sheet or private source. The committed summary should explain the order of operations and where to verify tables.

## Common Edge Cases

- Opposed traits can cancel, reduce, or transform sheet entries. Do this before optimizing points or buying extra XP.
- A normally negative trait can appear with positive XP from build choices; resolve it before final cleanup rather than leaving contradictory state.
- Partial negative traits are not automatically active. They only matter when the source procedure makes them reach an active negative level.
- Points-only Skill Field purchases require all included skills to be bought together and then adjusted by the source rebate.
- Mandatory language, English, and perception purchases can consume the last points in a points-only build; check them before declaring the sheet finished.
