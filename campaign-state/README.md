# Campaign State

Update these files during play. Keep them concise and current so GM mode can resume quickly.

Start with `campaign-state/setting-basics.md` for table canon, open campaign choices, faction posture, and starter hooks.

For live sessions, use:

- `current-campaign.md`: durable campaign assumptions.
- `current-mission.md`: active objective, scenes, stakes, and tactical handoff trigger.
- `player-characters.md`: PCs, test placeholders, conditions, and unresolved sheet details.
- `npc-roster.md`: named NPCs and current scene roles.
- `faction-roster.md`: confirmed faction pressures and attitudes.
- `session-log.md`: current and recent play notes.
- `rules-gaps.md`: missing or uncertain rules discovered during play.
- `playtest-bugs.md`: GM-mode workflow, routing, or usability problems.
