# Deterministic Mechanics Catalog

Status: issue `#71` planning catalog. This is not a rules engine and does not authorize broad resolver implementation.

Purpose: classify RPG procedures by owner, authority posture, automation readiness, expected helper contract, and next action before deterministic mechanic helpers are written.

## Source Boundary

- Use committed summaries, GM procedures, router metadata, manifest status, and page references as the public repository authority layer.
- Do not answer from mapped-only, source-lookup-only, private PDF text, copied tables, exact catalogs, unit records, or stat blocks.
- Preserve Classic BattleTech, MegaMek, MekHQ, and tabletop tactical authority when exact tactical combat or hard logistics matter.
- Deterministic helpers may calculate repeatable results only when route, authority, citation, and failure behavior are explicit.
- Helpers must emit proposed state changes rather than silently editing campaign files.

## Classification Vocabulary

- `agent-only`: the assistant/GM owns narrative framing, uncertainty handling, pacing, NPC intent, and temporary table rulings.
- `rules lookup only`: committed summaries and page references can guide a ruling, but the procedure is not ready for deterministic resolution.
- `authority gate`: a helper should first report whether a routed procedure can be adjudicated, needs private source lookup, needs external authority, or is blocked.
- `script prototype`: a narrow helper may be useful once the standard JSON contract and authority gate exist.
- `deterministic candidate`: the procedure has a stable enough shape for later repeatable helper logic.
- `state proposal`: the procedure may suggest campaign-file or pending-MekHQ updates, but must not apply them automatically.
- `external engine / MekHQ / MegaMek / tabletop authority`: exact resolution belongs outside MEK-RPG summaries or scripts.
- `blocked by source-review gap`: current committed material is insufficient for a usable ruling or helper without private source review.

## Adjudication Status

- `Adjudicable`: can be handled from committed summaries plus GM judgment, with citations and uncertainty preserved.
- `Provisional`: usable for table momentum as a narrow ruling, but should carry warnings or a follow-up note.
- `Private source lookup required`: committed material routes to source pages but should not reproduce or replace them.
- `External authority required`: use Classic BattleTech, MegaMek, MekHQ, or another table-selected authority.
- `Blocked`: do not adjudicate until source review, schema work, or ownership decisions are complete.

## Implementation Guardrails

- Early helpers should be PowerShell-friendly, JSON-in/JSON-out where appropriate, and fixture-tested through `scripts/test-all.ps1`.
- The catalog is intentionally conservative: `draft` summary status is enough for source-aware adjudication, not enough by itself for silent automation.
- Any helper that cannot prove route, authority, input values, and no-hidden-mutation behavior must return a warning, provisional status, source-lookup-required status, external-authority-required status, or blocked status.
